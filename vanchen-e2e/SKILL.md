# VanchEN E2E

使用官方 `$ego-browser` skill 操作真实页面。本 skill 只定义 VanchEN 的测试范围、工作流、断言和恢复规则；不要在此处复制或覆盖 ego-browser 的 API、选择器写法、脚本或版本兼容逻辑。

## 前提与范围

1. 读取仓库的 `docs/e2e/config.json`。文件不存在时，复制本 skill 的 `assets/config.json.template`，仅填入实际验证过的信息。
2. 选择 `active_environment`。用户指定其他环境时，仅使用 `enabled: true` 且 URL 非空的环境；访问范围限于环境 URL 的主机及配置中的 `allowed_hosts`。
3. 使用官方 skill 创建或复用与用户目标对应的 task space。遇到官方 skill 与运行时 API 不兼容、无法建立 browser 控制或登录态异常时，停止并报告环境问题；不要在本 skill 中编写 facade fallback。
4. 要求用户明确提供模块、目标菜单或测试目标、预期结果。缺少会改变测试范围的信息时先澄清；默认只执行查看、查询和页面可用性验证。

## 测试工作流

将需求整理为“模块 / 一级导航 / 分组 / 子菜单 / 操作与断言”，并保留实际观察到的完整路径。索引只用于辅助定位，真实 UI 才是运行事实。

1. 进入指定模块，并通过模块标题或等效 UI 状态确认切换成功。
2. 通过左侧导航定位目标一级菜单。除非用户明确要求测试模块搜索，否则禁止使用顶部“搜索模块”入口定位或进入业务菜单；悬浮菜单的经验见 [references/navigation-playbook.md](references/navigation-playbook.md)。
3. 仅在目标弹层可见、且弹层内存在目标分组和子菜单时点击。不要以页面主体中的同名文本、旧 ref 或旧坐标替代弹层证据。
4. 点击叶子菜单后，立即将鼠标移到目标页面内容区的安全空白处，确认悬浮弹层已关闭，再等待可观察的页面状态变化。普通页面优先采用官方 skill 的语义快照、locator 和当前轮 fresh ref。
5. 在微前端或 iframe 页面中，优先使用官方 skill 暴露的页面/iframe 语义能力；不要假定宿主页 DOM 查询能定位子应用控件。

## 页面断言

进入目标页面必须同时满足：

1. 目标名称的顶部 Tab 存在且为激活状态，或存在该模块等效的明确激活标识。
2. 页面出现与目标类型相符的业务特征。

列表页检查筛选、表头、数据行/空态或分页；详情/表单页检查字段和保存、提交、取消等动作入口；看板检查指标、图表或特有分类数据。只出现 Tab、空白页或加载骨架时，判定为页面断言失败。

## 操作授权

- 查询、查看列表和详情可按用户目标执行。
- 新增、编辑、删除、导入、导出、保存或提交，仅在用户明确授权了具体动作和目标数据时执行。
- 长操作或写操作触发后，不自动重复触发；超时后先报告潜在后端副作用。
- 需要最新详情数据时，返回列表后重新进入；不要假设已打开详情会自动刷新。

## 等待、无进展与恢复

以具体响应或具体 UI 状态为等待条件，例如加载状态消失、Tab 激活、表格数据变化、提示信息或按钮状态变化。不要仅以固定 sleep 或全局 network idle 判断成功。VanchEN 页面可能持续发送埋点、轮询或 tracking 请求；这些后台请求不代表页面仍在加载，也不得阻塞测试结束。

将 60 秒无有效进展作为软阈值。URL、激活 Tab、关键可见文本、加载状态、进度信息或截图发生预期变化，才算有效进展；重复等待、同一快照或同一截图不算进展。

出现明确运行证据（加载遮罩、spinner、禁用提交入口、进度或阶段文本）时，可短周期观察，整个观察流程最长 3 分钟。没有运行证据、运行证据消失但未达到预期，或达到总时限时，停止等待并按顺序恢复：

1. 采集页面信息、语义快照和失败证据，重新观察当前状态。
2. 从最近一个已验证的 UI 状态重新导航，必要时刷新当前 Tab；不得重复未授权写操作。
3. handoff 给用户或终止自动化并报告。

三次恢复后不得继续自动重试。不要将新建 task space 作为常规恢复手段；任务空间的所有权、接管和关闭均遵循官方 skill。用户接管后立即返回主会话，只有用户明确要求继续时才恢复。

## 结束与关闭

除非用户明确要求保留页面、需要用户在当前页面继续操作或当前任务已 handoff，否则关闭当前 task space 是 E2E 的必经最后步骤。将关闭作为独立的最终浏览器操作，并按官方 `$ego-browser` skill 确认关闭成功后再输出最终报告。

关闭失败或超时时，停止清理重试并报告“页面验证完成，但 task space 清理失败”；不要关闭整个 Ego 浏览器、使用全局进程终止方式，或将该任务视为完全完成。用户接管 task space 后不自动关闭或夺回控制。

## 证据、并发与维护

截图分为即时判断用的临时画面和可留档的最终证据。悬浮菜单、tooltip、加载态、通知、开发者工具或其他中间图层的截图只可用于当前步骤判断；放在 `/tmp`，完成判断后立即清理，不在报告或留档目录中保留。

只有页面已稳定、左侧悬浮菜单及其他遮挡图层已关闭时，才可保存关键成功证据。留档截图应完整展示目标业务页、激活 Tab 和足以证明断言的页面内容，不应包含悬浮菜单、顶部模块搜索、开发者工具或临时调试画面。失败需要截图时，也先尽量关闭与失败无关的悬浮图层，再记录实际失败状态。

临时截图使用 `/tmp`；只有用户明确要求留档时，才写入工作区内被 Git 忽略的证据目录。截图和报告不得包含 token、Cookie、Authorization 或不必要的账号和业务敏感信息。

并行验证不同菜单时，每个执行者使用独立 task space；写操作共享后端数据，必须使用唯一测试数据或串行执行。主会话应在约 90 秒未收到有效进展时要求执行者返回诊断，而不是无限等待。

仅主 agent 更新 `docs/e2e/config.json`。只记录真实页面中已验证的名称、层级和断言；不要记录 DOM class hash、坐标、等待秒数、用户通知状态或完整 selector。

## 输出

报告实际模块和完整路径、已执行操作、状态断言、证据路径、task space 清理结果，以及“通过”“失败”“超时恢复成功”“超时终止”或“验证通过，清理失败”。仅当页面验证通过且 task space 已成功关闭，或用户明确要求保留/接管页面时，才报告“通过”。不得将未执行、未验证或可能产生未确认后端副作用的操作称为通过。
