# VanchEN 页面导航经验

## 普通页面

将官方 `$ego-browser` skill 的语义快照、locator 和当前轮 fresh ref 作为默认路径。操作后立即重新观察并断言预期状态；不要跨轮复用 ref，也不要依赖上一次截图或旧坐标。

对列表页，先确认筛选区、操作入口、表头和数据行/空态，再执行查询或进入详情。选择业务记录前，先提取足够的结构化字段，避免仅凭大段快照中的同名文本做选择。

不要使用顶部“搜索模块”定位业务菜单，除非用户的测试目标就是该搜索功能。进入 FSMS 后，使用左侧一级导航及其弹层中的分组和叶子菜单，验证用户实际可见的菜单路径。

## 悬浮菜单

将悬停一级导航、确认弹层、定位弹层内目标并点击视为连续动作。点击前必须同时确认：一级导航可见、目标弹层可见、弹层内存在目标分组与子菜单、目标项具有可点击的可见区域。

点击叶子菜单后，立即将真实鼠标移到目标页面内容区的安全空白处，使弹层关闭；确认弹层不再遮挡页面后，才读取快照、截图或开始页面断言。不要固定坐标，按当前视口选择不包含按钮、表格行、输入框或浮层的空白区域。

不要将菜单悬浮状态作为关键流程截图留档。需要用截图确认弹层内容时，仅用于当前步骤判断并立即清理；进入目标页面且弹层关闭后，再保存完整页面证据。

旧弹层残留、侧栏高亮、索引中存在该路径或页面主体中出现同名文本，都不能替代弹层证据。弹层未出现时，先真实移出旧弹层，再重新观察；不要用合成鼠标事件猜测页面交互。

## 微前端与 iframe

FSMS 的菜单宿主与业务页面可能位于不同 DOM 上下文。宿主页的 DOM 查询不一定能定位业务页面控件；业务页面优先使用官方 skill 的 snapshot/ref 或 iframe 能力。只有官方语义能力无法表达目标时，才使用经过截图确认的视觉操作。

## 导航失败

将“弹层未出现”“菜单文本不匹配”“页面加载未完成”“页面断言失败”分别记录。首次失败后重新观察并尝试不同的已验证路径；第二次失败时保存证据，第三次恢复失败后 handoff 或终止。不要猜测 URL、点击正文同名元素或构造接口请求绕过真实页面。
