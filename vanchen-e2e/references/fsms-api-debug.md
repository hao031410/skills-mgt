# FSMS API 调试（按需读取）

仅当用户明确要求排查真实 UI 操作触发的接口、请求或响应体时使用。本参考不替代页面 E2E 工作流。

## 边界

- 先通过 UI 完成目标操作；不要手工重放或构造 HTTP 请求来代替点击。
- 不在输出、截图、文件或聊天中记录 Access-Token、Cookie、Authorization、完整鉴权 headers 或可识别的账号数据。
- 请求体和响应体可能含业务敏感数据；只提取解决当前问题所需的字段，并在报告中脱敏。
- 网络监听、response body 获取失败不应阻塞页面 E2E 的状态断言。

## Wujie 页面观察

FSMS 运行在微前端上下文中，宿主页面的 `window.fetch` hook 可能看不到子应用的真实请求。需要网络证据时，在触发 UI 操作前启用受控的 CDP Network 观察，并只记录脱敏后的 URL、方法、状态和必要业务字段。

response body 可能在请求完成后很快不可读取。将其视为可选调试证据，不要为获取它反复重试或改变页面测试结论。

## 与代码的关系

只有用户明确要求接口映射或实现排查时，才进入 `backend/fsms` 并先读取该子项目的 AGENTS.md / CLAUDE.md。代码分析用于解释 UI 现象，不替代真实页面验证。
