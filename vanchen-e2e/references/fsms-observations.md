# FSMS UI 实测观察

本文件记录当前 FSMS 页面中已经复现的用户可见现象，帮助测试时选择观察和断言方式；它不是选择器库，也不是自动化脚本。每次执行前仍须通过真实 UI 验证。

## 已验证导航现象

- FSMS 左侧一级导航可通过悬停展开按分组组织的子菜单。
- 目标子菜单打开后，顶部会出现对应 Tab；Tab 与页面业务特征可共同作为进入成功的证据。
- `档案 / 档案维护 / 参数配置` 打开的是含保存入口与多项配置字段的表单页。
- `档案 / 供应商审核 / 审核模板` 打开的是含关键字、审核类型、查询、新增、删除、表头和分页信息的列表页。
- 业务页面可能运行在微前端 iframe 内；通过语义快照获得的当前轮 ref 比宿主页 DOM 查询更可靠。
- 页面可能持续发送 tracking、埋点或轮询请求。以页面可见状态判断操作完成，不以全局 network idle 判断任务完成。
- 左侧菜单叶子项点击后，悬浮弹层可能保持可见；移动鼠标到业务内容区空白处后再进行断言和截图。
- 最终留档截图应展示弹层关闭后的完整业务页；菜单悬浮画面仅用于即时判断，不作为测试证据保留。

## 易变因素

菜单名称、分组层级、页面数据、登录态、通知、弹层容器、CSS class hash、坐标、动画时间和验证码均可能变化。记录新观察时附带验证日期和适用模块；只记录用户可见结构、成功断言和失效风险，不记录 token、Cookie、账号数据、固定坐标或完整 selector。
